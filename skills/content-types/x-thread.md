# X/Twitter Thread Format

**Version:** 1.0
**Last Updated:** 2025-12-27

---

## Purpose

Quick, punchy threads for build-in-public updates, hot takes, and real-time insights. Use X for more casual, contrarian, and experimental content.

---

## When to Use Thread Format

‚úÖ **Use for:**
- Build-in-public updates
- Hot takes and contrarian views
- Quick lessons from failures
- Live project updates
- Behind-the-scenes of a build
- Expanding a single insight into 5-10 tweets

‚ùå **Don't use for:**
- Long-form explanations (use LinkedIn or Substack)
- Professional positioning (use LinkedIn)
- Anything requiring visual frameworks (use carousel)

---

## Thread Length Guidelines

- **Minimum:** 3 tweets
- **Optimal:** 5-7 tweets
- **Maximum:** 10 tweets
- **Per tweet:** 1-3 sentences max

---

## Thread Structure Template

### Option 1: Build-in-Public Thread

```
Tweet 1: [Hook - What you're building/shipping]
Tweet 2: [Why you're building it]
Tweet 3: [What you tried first that didn't work]
Tweet 4: [What you're trying now]
Tweet 5: [Current status or early result]
Tweet 6: [Key lesson or insight]
Tweet 7: [CTA or question]
```

### Option 2: Hot Take Thread

```
Tweet 1: [Contrarian statement]
Tweet 2: [Why most people get this wrong]
Tweet 3: [Evidence or example]
Tweet 4: [The real insight]
Tweet 5: [What to do instead]
Tweet 6: [Conclusion or CTA]
```

### Option 3: Lesson Thread

```
Tweet 1: [Hook - What you learned the hard way]
Tweet 2: [The mistake or wrong assumption]
Tweet 3: [What happened as a result]
Tweet 4: [The turning point]
Tweet 5: [What you'd do differently]
Tweet 6: [Broader lesson]
Tweet 7: [Question to audience]
```

## Tone & Voice for X/Twitter

### More Casual Than LinkedIn:
- Shorter sentences
- More personality
- Less formal language
- More humor (if authentic)

### More Contrarian:
- Challenge conventional wisdom
- Hot takes are encouraged
- Be willing to be wrong publicly

### More Transparent:
- Share failures openly
- Show works-in-progress
- Admit when you don't know something

---

## Character Count Strategy

### Optimal Length Per Tweet:
- **Under 280 characters** is ideal (fits in one view)
- **240 characters or less** is even better (easier to read)
- Break long thoughts into multiple tweets

### Formatting Tips:
- Use line breaks liberally
- One idea per tweet
- White space improves readability

## When to Thread vs. Single Tweet

### Use Thread When:
- Story needs context (3+ connected ideas)
- Build-in-public update with steps
- Hot take that requires explanation

### Use Single Tweet When:
- Quick insight stands alone
- Simple observation
- Sharing a link to longer content

---

## Visual Content for Threads

### What Works on X:
- Screenshots of builds
- Simple charts or graphs
- Before/after comparisons
- Code snippets (syntax highlighted)
- Personal photos (sparingly)

### What Doesn't Work:
- Heavy carousel-style PDFs (use LinkedIn)
- Walls of text as images
- Overly designed graphics
